PHP中获取当前页面的完整URL
发表于 2016-06-22| 更新于 2019-11-21|php
阅读量:

$_SERVER 是一个包含了诸如头信息(header)、路径(path)、以及脚本位置(script locations)等等信息的数组。这个数组中的项目由 Web 服务器创建。

<?php
#测试网址:     http://www.dou520.wang/blog/testurl.php?id=5


//获取域名或主机地址 
echo $_SERVER['HTTP_HOST']."<br>"; #www.dou520.wang


//获取网页地址 
echo $_SERVER['PHP_SELF']."<br>"; #/blog/testurl.php


//获取网址参数 
echo $_SERVER["QUERY_STRING"]."<br>"; #id=5


//获取用户代理 
echo $_SERVER['HTTP_REFERER']."<br>"; 


//获取完整的url
echo 'http://'.$_SERVER['HTTP_HOST'].$_SERVER['REQUEST_URI'];
echo 'http://'.$_SERVER['HTTP_HOST'].$_SERVER['PHP_SELF'].'?'.$_SERVER['QUERY_STRING'];
#http://www.dou520.wang/blog/testurl.php?id=5


//包含端口号的完整url
echo 'http://'.$_SERVER['SERVER_NAME'].':'.$_SERVER["SERVER_PORT"].$_SERVER["REQUEST_URI"]; 
#http://www.dou520.wang:80/blog/testurl.php?id=5


//只取路径
$url='http://'.$_SERVER['SERVER_NAME'].$_SERVER["REQUEST_URI"]; 
echo dirname($url);
#http://www.dou520.wang/blog
?>

重点说下HTTP_HOST 和 SERVER_NAME 的区别。


相同点：
当满足以下三个条件时，两者会输出相同信息。
1. 服务器为80端口
2. apache的conf中ServerName设置正确
3. HTTP/1.1协议规范


不同点：
1. 通常情况：
_SERVER["HTTP_HOST"] 在HTTP/1.1协议规范下，会根据客户端的HTTP请求输出信息。
_SERVER["SERVER_NAME"]
默认情况下直接输出apache的配置文件httpd.conf中的ServerName值。


2. 当服务器为非80端口时：
_SERVER["HTTP_HOST"] 会输出端口号，例如：aaa.com:8080
_SERVER["SERVER_NAME"] 会直接输出ServerName值
因此在这种情况下，可以理解为：HTTP_HOST = SERVER_NAME : SERVER_PORT


3. 当配置文件httpd.conf中的ServerName与HTTP/1.0请求的域名不一致时：
httpd.conf配置如下：
<virtualhost *>
ServerName aaa.com
ServerAlias www.aaa.com
</virtualhost>
客户端访问域名www.aaa.com
_SERVER["HTTP_HOST"] 输出 www.aaa.com
_SERVER["SERVER_NAME"] 输出 aaa.com

所以，在实际程序中，应尽量使用_SERVER["HTTP_HOST"] ，比较保险和可靠。

需要注意的是，端口是可以改变的，前几天工作中就有个网站因为端口改变，而图片路径写入用的是_SERVER["HTTP_HOST"]，导致图片不能正常显示。
