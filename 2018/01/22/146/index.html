xdebug断点调试安装

安装准备：

  1. PHP环境（我使用的是nginx1.12+php7）

  2. PHPStorm

1. 下载dll扩展放到PHP目录下ext中。

2. 修改php.ini

;注意是zend_extension
zend_extension=php_xdebug.dll
;远程地址,指的是连接本php所在服务器的客户端ip
xdebug.remote_host=127.0.0.1
;远程端口，不要冲突
xdebug.remote_port=9999
xdebug.remote_handler=dbgp
xdebug.collect_vars = On
xdebug.collect_return = On
xdebug.collect_params = On
;编辑器名称
xdebug.idekey=phpstorm
xdebug.profiler_output_dir="D:\wnmp\php-7.0.26    mp�bug" 
;设定函数调用监测信息的输出文件的路径
xdebug.trace_output_dir="D:\wnmp\php-7.0.26    mp�bug"
xdebug.remote_enable =1xdebug.remote_mode = req
xdebug.profiler_enable = off
xdebug.profiler_enable_trigger = off
;开启远程调试自动
xdebug.remote_autostart = On

配置成功，在phpinfo页面会出现xdebug版本和idekey。

3. PHPStorm配置

settings:

> Languages & Frameworks → PHP 
配置对应的interpreter，其实就是设置下PHP executable路径
如：D:\wnmp\php-7.0.26\php.exe


> Languages & Frameworks → PHP → Debug 
修改端口


> Languages & Frameworks → PHP → Debug → DBGp Proxy
就是上面php设置的idekey，host，port。


> Languages & Frameworks → PHP → Servers
添加选择Xdebug

完成后

Run → Edit Configuration
添加一个PHP web application配置，选择刚才配置的Servers

这样就完成了！

run，debug都能直接用起来了。
